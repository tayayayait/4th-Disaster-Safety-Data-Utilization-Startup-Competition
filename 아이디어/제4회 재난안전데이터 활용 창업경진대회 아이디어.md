좋아요. 이 대회는 단순한 “재난 알림 앱”보다 **AI가 데이터를 근거로 위험을 판단하고, 지도 위에서 실행 가능한 조치까지 제안하는 서비스**가 훨씬 유리해 보입니다. 공고문상 제품·서비스 개발 부문은 접수 마감일까지 시제품이 완료되어야 하고, 심사 기준도 데이터 활용성, 창의성, 실현 가능성, 창업 연계성, AI 프롬프트 엔지니어링 활용 능력을 봅니다. 안내자료에서도 재난 유형은 자유롭게 선택 가능하고, 실데이터와 가상데이터를 함께 쓸 수 있으며, 지도 표출·Agent 개발·API 연계가 있으면 서비스 개발 공모에 적합하다고 설명합니다.

기존 국민안전24 같은 서비스는 재난상황정보, 재난문자, 재난안전뉴스 등을 제공하는 정보 전달형 성격이 강합니다. ([Safe Korea][1]) 그래서 이번 아이디어는 **“재난 발생을 알려주는 서비스”가 아니라, AI가 ‘어디가 위험한지, 누구를 먼저 대피시킬지, 어떤 CCTV·계측기·대피소·도로를 확인할지’를 판단해주는 실행형 서비스**로 잡는 게 좋습니다.

## 1위 후보. **SafeRoute Ops AI — 침수·산사태 현장조치 AI 지휘관**

### 해결 문제

집중호우 때 재난문자나 기상특보는 오지만, 담당자나 시민 입장에서는 **“우리 동네에서 어디가 제일 위험한지, 어느 도로를 막아야 하는지, 누구를 먼저 대피시켜야 하는지”**를 바로 알기 어렵습니다.

- AI가 위험지역 TOP 5, 대피 대상, 접근 차단 위치, 확인할 CCTV·계측기, 담당기관 연락 순서 제안
- 시민용 화면: “현재 위치 기준 안전 대피 경로”

- 출력 형식은 JSON 고정: `위험단계 / 위험지역 / 대피대상 / 차단위치 / 확인장비 / 알림문안`
- AI 답변에는 반드시 근거 데이터명과 신뢰도 표시

### 차별점

- 프론트: React + Leaflet 또는 MapLibre

- AI: Gemini 또는 GPT API
- 실시간 API: 기상청 초단기실황
- 지도 레이어: 침수흔적도, 산사태위험지도, 대피소, 교통통제
- 데모 시나리오: “시간당 50mm, 누적 150mm” 입력 → AI가 위험지도와 조치카드 자동 생성

- AI가 작업중지 권고, 현장점검 순서, 작업자 안내문 생성
- 관리자용 조치 이력 저장 및 보고서 자동 생성

### 활용 데이터·API

- 기상청 초단기실황·단기예보
- 침수흔적도
- 산사태위험지도
- 경찰청 교통돌발정보
- 브이월드 지도 API
- 사업장 자체 입력 데이터: 현장 위치, 장비, 위험시설, 작업자 수

### AI 활용 방식

- 공공데이터 + 사업장 자체 위험요소를 결합해 위험등급 산정
- AI Agent가 “현재 기상 조건에서 이 현장이 해야 할 일”을 체크리스트로 제시
- 현장 사진 업로드 시 위험요소 설명 기능은 MVP 이후 확장

### 차별점

일반 재난안전 앱은 시민용이 많지만, 이 서비스는 **기업의 안전보건·재난대응 업무**에 바로 연결됩니다.

### MVP 구현 방법

- 공사장 3~5개를 가상 등록
- 기상 API와 지도만 연동해도 시연 가능
- 현장별 위험카드, 체크리스트, 알림문안 자동 생성
- 보고서 PDF 출력 기능까지 넣으면 제품성이 강해짐

### 사업화 가능성

- 건설사, 산업단지, 물류센터, 발전소, 공장 대상 B2B
- 안전관리 SaaS 월 구독 모델
- 중대재해 예방, 재난대응 매뉴얼 디지털화 시장과 연결 가능
