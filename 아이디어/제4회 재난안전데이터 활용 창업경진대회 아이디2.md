## 1. 공고문 분석: 심사위원이 원하는 결과물

이번 대회의 **제품 및 서비스 개발 부문**은 단순한 아이디어나 프롬프트 문구를 제출하는 부문이 아닙니다. **재난안전데이터를 활용한 웹·앱 시제품이 접수 마감일까지 실제로 작동하는 상태**여야 하며, 1차 심사에서는 데이터 활용성·창의성·실현 가능성·창업 연계성·AI 프롬프트 엔지니어링 활용 능력을 평가합니다.

안내자료 1쪽에서는 상용 LLM에 데이터를 넣고 질의응답만 확인한 결과는 아이디어 부문에 가깝고, **지도 표출·AI Agent·API 연계 같은 추가 개발 요소가 있어야 서비스 개발 부문에 적합하다**고 구분합니다. 실제 데이터가 부족할 때는 가상데이터로 보완하는 것도 허용됩니다.

따라서 수상 가능성이 높은 구조는 다음과 같습니다.

> **공공데이터 수집 → 위험도 계산 → AI Agent의 조치 추천 → 지도 시각화 → 담당자·시민 알림 → 조치 이력 저장**

또한 LLM이 위험도를 임의로 판단하게 하기보다는, 별도의 규칙·통계·예측 엔진이 위험점수를 계산하고 **LLM은 계산 결과를 설명하고 대응 조치로 변환하는 역할**을 맡기는 것이 안전성과 신뢰성 면에서 유리합니다.

# 재난안전 AI 프로그램 아이디어 TOP 5

## 1위. 세이프셸터 트윈

### AI 기반 대피소 적합도·혼잡도 예측 및 분산대피 서비스

**한 줄 설명**

> 재난 발생 시 가장 가까운 대피소가 아니라, 현재 위험과 이동 가능성, 수용 여력, 이용자 특성을 분석해 가장 안전한 대피소와 경로를 추천하는 AI 서비스

### 해결하려는 문제

재난 발생 시 대피소 위치를 알고 있더라도 다음 문제는 해결되지 않습니다.

- 해당 대피소까지 가는 길이 침수되었는가
- 이미 수용인원을 초과했는가
- 지하 대피소가 현재 재난 유형에 적합한가
- 휠체어·고령자·영유아가 이용할 수 있는가
- 여러 대피소 중 주민을 어떻게 분산시켜야 하는가

### 주요 기능

1. **재난별 대피소 적합도 계산**

   - 침수 시 지하 대피소 감점
   - 지진 시 옥외대피장소 우선
   - 산불 시 풍하측 대피소 제외
   - 해일 시 해발고도와 이동시간 반영

2. **AI 안전대피 경로 추천**

   - 최단거리 대신 침수·산사태·도로통제 구간을 피해 경로 산출
   - 일반인, 고령자, 휠체어 이용자별 경로 분리

3. **대피소 혼잡도 예측**

   - 대피소 수용인원, 주변 인구, 이동시간을 이용해 10분·30분 후 예상 혼잡도 계산
   - 과밀 예상 시 다른 대피소로 자동 분산

4. **AI 재난대응 Agent**

   - 시민에게는 “어디로, 어느 길로, 무엇을 챙겨 이동할지” 안내
   - 담당 공무원에게는 “어느 대피소를 개방하고 어느 지역 주민을 배정할지” 제안

5. **실시간 알림**

   - 추천 대피소 변경
   - 도로 통제 발생
   - 대피소 혼잡도 상승
   - 위험단계 상향 시 푸시 알림

### 활용 데이터·API

- 행정안전부 민방위대피시설 조회서비스: 위치, 시설면적, 최대 수용인원, 운영상태
- 지진옥외대피장소·지진해일 긴급대피장소
- 기상청 초단기실황·초단기예보·단기예보
- 도시침수지도·홍수위험지도·산사태위험지도
- 행정안전부 긴급재난문자
- 국토교통부 교통소통정보·돌발상황정보
- 행정동별 연령별 인구
- 장애인 편의시설 데이터
- VWorld 공간정보·지도 API

민방위대피시설 API에는 시설 규모와 최대 수용인원, 운영상태, 위치정보가 포함되어 있고, 도시침수지도는 침수심을 5단계로 구분해 제공합니다. 도로 돌발상황 API에는 도로침수·기상악화·통제 여부 등이 포함되므로 대피경로 계산에 활용할 수 있습니다. ([데이터.go.kr][1])

### AI 활용 방식

- **위험도 모델:** 재난강도 × 경로위험 × 대피소 적합성 × 예상 혼잡도
- **최적화 알고리즘:** 주민과 대피소를 연결하는 최소비용 흐름 또는 용량제약 최적화
- **AI Agent:** API를 호출해 최신 데이터를 확인한 뒤 행동요령 생성
- **RAG:** 국민행동요령, 지자체 매뉴얼, 대피소 운영규칙을 검색해 근거와 함께 답변
- **가상데이터:** 실시간 대피소 입장 인원은 QR 체크인 또는 시뮬레이션 데이터로 구현

### 차별점

기존의 위치 안내 방식과 달리 **“대피소 찾기”가 아니라 “재난 유형별 대피소 배정과 주민 분산”**을 해결합니다.

특히 시민용 서비스와 담당자용 관제 화면을 동시에 만들 수 있어, 공익성과 B2G 사업성을 함께 보여줄 수 있습니다.

### MVP 구현 방법

대상 지역을 한 개 시·군·구로 제한하고, 재난 유형도 **집중호우·도시침수 하나만** 선택합니다.

- 대피소 20~50곳 표시
- 실제 기상 API 연동
- 도시침수 위험 레이어 표시
- 가상 주민 300명과 가상 대피소 점유율 생성
- 안전점수 상위 대피소 3곳 추천
- 도로통제 발생 시 경로 자동 재계산
- 담당자용 대피소 혼잡도 관제 화면
- 시민용 PWA 푸시 알림

### 사업화 가능성

- 지자체 재난안전상황실용 SaaS
- 스마트시티 통합플랫폼 연계
- 대형 사업장·산업단지·대학·축제장의 대피관리 서비스
- 보험사·시설관리기업의 재난 리스크 관리 솔루션
- 대피소 실시간 수용현황 데이터 구축 사업

## 2위. 레스큐옵스 AI

### 재난자원 선배치·긴급조달 의사결정 코파일럿

**한 줄 설명**

> 호우·폭설·산불 발생 전에 필요한 장비와 인력을 예측하고, 어디에 배치하며 어느 업체에 연락해야 하는지까지 추천하는 담당 공무원용 AI Agent

### 해결하려는 문제

재난담당자는 기상특보가 발표되면 짧은 시간 안에 다음을 결정해야 합니다.

- 어느 지역에 양수기·차수판·제설차를 몇 대 배치할지
- 어느 도로를 먼저 통제할지
- 어떤 용역업체와 장비업체에 연락할지
- 과거 계약 단가와 수행 실적이 어땠는지
- 담당 부서별로 어떤 임무를 배정할지

현재 데이터를 각각 확인하더라도 이를 하나의 실행계획으로 만드는 데 시간이 소요됩니다.

### 주요 기능

- 6시간·12시간·24시간 후 읍면동별 위험도 예측
- 필요한 장비·인력 수량 추정
- 장비 선배치 위치 추천
- 예상 침수지점과 우선 통제도로 지도 표시
- 과거 지자체 계약 업체·계약금액·계약내용 검색
- AI가 비상연락망과 임무배정표 생성
- 위험단계가 바뀌면 조치계획 자동 갱신
- 담당자가 승인한 조치의 이력 관리

### 활용 데이터·API

- 기상청 초단기·단기예보
- 홍수위험지도·도시침수지도·산사태위험지도
- 산사태 예보발령 정보
- 행정동별 인구와 사회복지시설
- 국토교통부 교통소통·돌발상황정보
- 행정안전부 지방재정365 계약현황
- 지자체 장비·인력 현황은 가상데이터 또는 직접 등록

지방재정365 계약현황 API에는 계약명, 계약일자, 계약금액, 업체명 등이 포함되어 있어 과거 재난 관련 용역업체와 조달 이력을 검색하는 기능에 활용할 수 있습니다. ([데이터.go.kr][2])

### AI 활용 방식

- 과거 강우량과 피해·조치 데이터를 활용한 자원수요 예측
- 위험지역과 장비기지 간 거리 기반 배치 최적화
- 계약 데이터 RAG를 통한 업체·단가 검색
- AI Agent가 부서별 조치계획, 전화 순서, 현장 체크리스트 작성
- 실제 데이터가 없는 자원 수량은 가상데이터로 시나리오화

### 차별점

단순히 “위험하다”고 알려주는 것이 아니라 다음의 전체 과정을 연결합니다.

> **예보 → 위험지역 → 필요한 자원 → 배치 위치 → 공급업체 → 임무지시**

공고 안내자료의 집중호우 대응 예시와 잘 맞으면서도, **긴급조달과 자원 최적화**를 결합해 차별화할 수 있습니다.

### MVP 구현 방법

- 한 개 구청의 집중호우 상황으로 제한
- 양수기, 차수판, 순찰차량, 공무원, 민간업체 등 5종 자원
- 과거 계약 데이터 20~50건 수집
- 가상 장비 현황 30건 생성
- 3개 강우 시나리오에 따라 자원 배치결과 비교
- AI가 “현재 해야 할 일 TOP 10” 생성
- 지도에서 배치 전·후 위험도 변화 시각화

### 사업화 가능성

- 지자체 재난안전부서
- 도로·철도·공항·항만 공공기관
- 대형 건설현장과 산업단지
- 재난관리자원·조달 시스템 연계
- 지자체별 구독형 재난대응 코파일럿

**한 줄 설명**

- 냉방시설 접근성
- 무더위쉼터까지의 거리
- 대기오염

- 시간대별 체감온도
- 담당 인력의 방문 가능 시간

개인별 건강정보를 사용하지 않더라도, 지역 단위 데이터를 활용해 우선 점검지역을 선정할 수 있습니다.

### 주요 기능

- 행정동·격자별 폭염 취약도 지도
- 무더위쉼터 수용 가능 인원과 접근성 분석

- 복지담당자 방문 경로 최적화

- 미확인 시설 재알림
- 시민에게 가장 가까운 것이 아닌 가장 적합한 쉼터 추천
- 폭염·오존·미세먼지 복합위험 알림

### 활용 데이터·API

- 기상청 생활기상지수
- 기상청 초단기·단기예보

- 행정동별 연령별 인구
- 전국사회복지시설 표준데이터
- 장애인 편의시설 데이터
- 교통·지도 API

무더위쉼터 데이터에는 면적, 이용가능인원, 에어컨·선풍기 보유 정보가 포함되며, 생활기상지수와 에어코리아 API를 함께 사용하면 기상과 대기질을 결합한 복합위험도를 만들 수 있습니다. ([데이터.go.kr][3])

## 4위. 윈드펜스 AI

### 산불 확산방향 예측 및 마을별 선제대피 서비스

**한 줄 설명**

> 산불 발생지와 실시간 풍향·풍속·지형을 분석해 1~3시간 후 확산 가능 구역과 우선대피 마을, 진화자원 배치지점을 추천하는 AI 서비스

### 해결하려는 문제

산불 발생 사실을 알아도 주민과 담당자는 다음을 즉시 판단하기 어렵습니다.

- 불이 어느 방향으로 번질 가능성이 높은가
- 어느 마을을 먼저 대피시켜야 하는가
- 대피경로가 불의 진행 방향과 겹치지 않는가
- 진화대원과 소화시설은 어디에 배치되어 있는가
- 풍향이 바뀌면 기존 대피경로를 계속 사용해도 되는가

### 주요 기능

- 풍향·풍속에 따른 산불 확산 부채꼴 표시
- 30분·1시간·3시간 예상 위험구역 애니메이션
- 마을별 예상 도달시간과 대피 우선순위
- 풍하측 대피소 자동 제외
- 안전대피 경로와 도로통제 지점 추천
- 산불 진화대원 대기장소·소화시설 표시
- 풍향 변화 시 경로 재계산 및 긴급알림
- AI 현장지휘 Agent가 임무와 연락 순서 생성

### 활용 데이터·API

- 산림청 산불발생통계와 산불발생이력
- 산불 진화대원 대기장소
- 산불 소화시설 위치
- 기상청 풍향·풍속·습도·강수예보
- 행정동 인구와 건축물 분포
- 대피소와 도로 돌발상황
- VWorld 지형·공간정보

산림청 데이터에는 산불 발생 위치·시간·원인·피해 규모 등이 포함되며, 진화대원 대기장소와 산불소화시설 위치 데이터도 별도로 제공되고 있어 시범 관제지도를 구성할 수 있습니다. ([데이터.go.kr][4])

### AI 활용 방식

- 간소화된 셀룰러 오토마타 또는 격자 확산모델
- 풍향·풍속·경사도·산림 인접도를 이용한 확산위험 계산
- 마을별 대피 우선순위 예측
- AI Agent가 대피·통제·진화자원 배치 조치 작성
- 실제 산불 대신 가상의 발화지점을 선택해 시뮬레이션 가능

### 차별점

산불 발생정보를 표시하는 수준을 넘어 **시간에 따라 움직이는 예상 위험구역과 대피결정**을 연결합니다. 발표 시 지도 애니메이션 효과가 커서 시각적 전달력이 높습니다.

### MVP 구현 방법

- 산불 위험이 높은 한 개 군 단위로 제한
- 발화지점을 지도에서 선택
- 기상 API의 현재 풍향·풍속 적용
- 30분 간격 확산구역 생성
- 5개 마을의 대피 우선순위 표시
- 가상 풍향 변경 버튼으로 경로 변경 시연

### 사업화 가능성

- 산림청·지자체 산림부서
- 산불상황실과 소방기관
- 국립공원·휴양림·캠핑장 운영기관
- 산림 인접 산업시설과 문화재 관리기관
- 지역 단위 산불대피훈련 시뮬레이터

## 5위. 리스크도어 AI

### 건물·사업장별 재난 행동계획 자동생성 서비스

**한 줄 설명**

> 주소 하나를 입력하면 건축물과 주변 재난위험을 분석해 해당 건물에 맞는 사전점검, 영업중단, 대피, 시설보호 조치를 생성하는 AI 안전관리 서비스

### 해결하려는 문제

같은 지역에 같은 재난문자가 발송되더라도 건물마다 대응방법은 달라집니다.

- 지하층이 있는 상점
- 노후 저층주택
- 요양시설
- 어린이집
- 공장과 물류창고
- 산지와 인접한 숙박시설

그러나 소규모 시설은 자체 재난안전 담당자나 전문 매뉴얼이 없는 경우가 많습니다.

### 주요 기능

- 주소 입력 시 건물 기본정보와 주변 위험 조회
- 지하층·용도·층수·사용승인연도 기반 취약성 분석
- 재난별 30분·3시간·24시간 행동카드 생성
- 직원 출근중지·영업중단 권고 기준 설정
- 시설물 이동·전원차단·출입통제 체크리스트
- 주변 대피소와 안전경로 표시
- 관리자 확인 여부와 조치 이력 저장
- 위험단계 상승 시 자동 알림

### 활용 데이터·API

- 국토교통부 건축HUB 건축물대장정보
- 도시침수·홍수·산사태위험지도
- 기상청 예보
- 긴급재난문자
- 대피소·교통 돌발상황
- 사회복지시설·어린이집·병원 등 시설 데이터
- VWorld 지도

건축HUB API에서는 건물 용도, 구조, 층수, 사용승인일, 세대수, 내진설계 관련 정보 등을 조회할 수 있어 건물 단위 취약성 분석의 기초자료로 활용할 수 있습니다. ([데이터.go.kr][5])

### AI 활용 방식

- 건물 특성과 재난위험을 결합한 규칙 기반 위험등급
- 업종·시설유형별 대응매뉴얼 RAG
- AI Agent가 시간순 행동계획 생성
- 관리자 질문에 건물정보와 재난상황을 근거로 답변
- 향후 평면도 이미지 분석 기능 확장 가능

### 차별점

지역 단위 재난정보를 **건물과 사업장 단위의 실행계획**으로 변환합니다. 특히 소상공인·학교·복지시설을 대상으로 한 B2B·B2G 모델이 명확합니다.

### MVP 구현 방법

- 집중호우와 도시침수만 지원
- 가상 또는 실제 건물 10곳 등록
- 건축물대장과 침수위험 데이터 결합
- 건물별 위험등급과 행동카드 생성
- 시설관리자용 체크리스트와 완료 버튼 구현
- 조치결과 PDF 보고서 생성

### 사업화 가능성

- 소상공인·프랜차이즈 본사
- 학교·어린이집·요양시설
- 공동주택과 시설관리업체
- 산업단지·물류센터
- 보험사의 사업장 위험관리 서비스

**리스크도어 AI**는 사업화가 명확하지만 여러 시설유형과 재난을 동시에 지원하면 범위가 커집니다. 초기에는 지하상가·요양시설 등 한 가지 시설군으로 좁히는 것이 좋습니다.

# 최종 선정 1위: 세이프셸터 트윈

## 선정 이유

### 1. 문제를 한 문장으로 설명할 수 있음

> “재난 시 가장 가까운 대피소가 아니라, 지금 갈 수 있고 들어갈 수 있으며 내 조건에 맞는 대피소를 찾아주는 서비스”

심사위원이 즉시 이해할 수 있습니다.

### 2. 공공데이터 융합이 명확함

대피소 하나만 사용하는 것이 아니라 다음 데이터를 함께 사용합니다.

- 대피소 수용인원
- 기상예보
- 침수·산사태 위험
- 도로통제
- 인구구조
- 장애인 편의시설
- 긴급재난문자

계획서의 “데이터 간 융합을 통한 아이디어 발굴 정도” 항목을 설득하기 좋습니다.

### 3. 대회가 요구하는 기술을 모두 보여줄 수 있음

- 지도 시각화
- 외부 API
- AI Agent
- 위험도 예측
- 경로 탐색
- 실시간 알림
- 가상데이터 활용
- 담당자용 관제화면

### 4. MVP 범위를 통제하기 쉬움

전국 모든 재난을 구현할 필요가 없습니다.

**집중호우 + 한 개 지역 + 대피소 30곳 + 가상 주민·혼잡도**만으로도 완성도 높은 시연이 가능합니다.

### 5. 데이터가 없는 부분도 사업기회가 됨

현재 공개된 대피소 데이터에는 시설 규모와 최대 수용인원이 있지만, 실시간 입장 인원이나 반려동물 동반 여부, 휠체어 접근 가능성 등은 충분하지 않을 수 있습니다. 이를 다음과 같이 제안할 수 있습니다.

- QR 기반 실시간 입장인원 수집
- 지자체 관리자용 대피소 상태 변경
- 대피소 편의·접근성 항목 추가 개방
- 혼잡도 표준 API 구축

이는 공모작 계획서의 **공공데이터 추가개방 아이디어**에도 연결됩니다.

## 권장 MVP 화면 4개

1. **시민용 재난지도**

   - 현재 위치
   - 침수위험 구역
   - 추천 대피소 1~3순위
   - 안전대피 경로

2. **AI 행동안내 화면**

   - 현재 위험단계
   - 대피 시점
   - 챙겨야 할 물품
   - 피해야 할 도로
   - 추천 근거와 데이터 기준시각

3. **담당자 관제화면**

   - 대피소별 수용인원·예상 혼잡도
   - 주민 유입 예상량
   - 과밀 대피소
   - 추가 개방이 필요한 시설

4. **재난 시뮬레이션 화면**

   - 시간당 강우량 변경
   - 도로통제 설정
   - 대피소 폐쇄 설정
   - 변경 전후 주민 배정 결과 비교

## 권장 개발 구성

- **프런트엔드:** Next.js 또는 React PWA
- **지도:** MapLibre GL + VWorld·공공 WMS 레이어
- **백엔드:** FastAPI
- **공간 데이터베이스:** PostgreSQL + PostGIS
- **위험도 계산:** Python 규칙 엔진 + NetworkX 또는 OR-Tools
